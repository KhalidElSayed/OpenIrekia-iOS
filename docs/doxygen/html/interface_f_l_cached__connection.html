Detailed Description

Improvement over the FLRemote_connection class.

For the FLCached_connection, the contents of URLs are stored in memory and reused from there.

- (void) request: (NSString*)  url
cache_token: (int)  cache_token 

Request an URL.

However, unlike FLRemote_connection, this will first check the memory cache, for which you are required to pass a cache_token, which is really just the internal identifier associated with the tab this request is happening from.

Note that setting dont_cache to YES before calling this will ignore all caches, memory and disk.
