Detailed Description

Handles the photo gallery view.

The gallery view is implemented not with a scroll view, but with a table view. This is slightly annoying, as it will cause problems in the future with landscape rotation, but for the moment it works. The view puts FLGallery_cell items in the table, which actually handle multiple thumbs.

The items are expected to be a list of dictionaries from which FLGallery_item objects can be created. If they cannot, they will be ignored.

- (void) switch_item: (int)  direction [implementation]

Get notifications from FLStrip_view_controller about scroll changes.

Note that unlike the FLNews_view_controller, the control is inverted in this class! Yes, the switch_item doesn't actually switch the item. It is only used for the child (the strip controller) to tell the parent the item has changed. Otherwise, the strip is in charge of doing everything.

So, this method is called when the page changed, but we figure out the index position in the list obtaining the selected item. That's much saner than trying to figure out what cell of the row was pressed.
